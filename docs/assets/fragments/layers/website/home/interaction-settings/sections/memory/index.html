Memory

Control how the agent uses project context, session continuity, and workspace memory during interaction

Mode

Session only Use temporary context for the current interaction without long-range project binding
Workspace memory Preserve context across workspace sessions, project decisions, and agent continuity records
Governed memory Use approved planning documents, changelogs, and institutional records as the memory source

Controls

Project memory Allow the agent to use active project context while planning and responding
Decision memory Retain confirmed strategic and implementation decisions for future sessions
Transient context only Keep the current command isolated when sensitive or exploratory work should not persist

Source

Memory source Workspace + governed records
Reset session context Manual approval required
