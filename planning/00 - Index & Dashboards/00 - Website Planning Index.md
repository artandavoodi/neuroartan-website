## 4. Current Strategic Decision

The current strategic direction is to prioritize the website/web app layer as the primary development continuity surface for the next thirty days.

The software layer remains strategically important but is not the critical path for this cycle.

The AI Development Cockpit should become the internal command surface for prompt composition, terminal scan generation, exact file targeting, patch instruction generation, provider routing, GitHub commit message support, and daily execution control.

---

## 5. Current Execution Priority

Current priority:

1. preserve the existing command-generation cockpit frontend
2. complete Developer Mode architecture documentation before runtime work
3. keep provider, GitHub, sandbox, internet egress, patch, test, commit, and pull request execution server-side
4. prohibit frontend secrets and browser-side repository mutation
5. define backend interface contracts before implementing runtime code
6. proceed to read-only backend stubs only after architecture review

## 7. Codex Execution Control

Codex must be used through the Codex Daily Prompt Queue.

Codex should not receive broad redesign instructions.

Each Codex prompt must be scoped to one bounded objective and must request:

- created files
- modified files
- unrelated files confirmation
- verification command
- completion summary
