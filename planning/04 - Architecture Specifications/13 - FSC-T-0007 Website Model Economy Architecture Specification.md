# FSC-T-0007 Website Model Economy Architecture Specification

---

## I. Purpose

This specification defines the website-layer architecture required to support FSC-T-0007.

The website must represent ICOS as a governed model universe without prematurely launching marketplace, payout, autonomous hiring, professional-domain, or posthumous-economy systems.

---

## II. Architecture Principle

The website must reuse the existing modular structure:

- `docs/assets/data` for structured records.
- `docs/assets/js/layers/website/system` for product/system state.
- `docs/assets/js/layers/website/sections` for public content surfaces.
- `docs/assets/css/layers/website` for token-aligned styling.
- `planning` for governance, architecture, routing, and public-positioning control.

No duplicate product reality may be created.

No FSC-T-0007 feature may bypass existing profile, model, entitlement, permission, dashboard, registry, or system layers.

---

## III. Required Website Architecture Branches

1. Model Economy data layer.
2. Model Birth Identity data layer.
3. Public / private model identity boundary.
4. Model Dignity public explanation and owner-control boundary.
5. Marketplace-readiness schema.
6. Monetization and hiring state schema.
7. Inter-model hiring readiness boundary.
8. Ocean-Brain / model population awareness data layer.
9. Public-positioning safety gate.
10. Implementation-routing document control.

---

## IV. Implementation Priority

1. Planning documents.
2. Data schemas.
3. JS state modules.
4. CSS system files.
5. Orchestrator imports.
6. Public section copy.
7. Profile/private dashboard surfaces.
8. Marketplace concept pages only after review.
