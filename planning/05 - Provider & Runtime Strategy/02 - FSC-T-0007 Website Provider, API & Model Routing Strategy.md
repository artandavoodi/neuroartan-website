# FSC-T-0007 Website Provider, API & Model Routing Strategy

---

## I. Purpose

This strategy defines how website-facing model economy surfaces must treat API-provider embedding, provider routing, and runtime state.

---

## II. Rule

Provider and API routing are private operational facts.

Public surfaces may show only approved trust or capability signals.

Private owner-facing surfaces may show controlled operational state.

---

## III. Website Boundary

The website must not expose:

- private model ID;
- private serial identity;
- provider route;
- API route;
- token authority;
- source authorization evidence;
- private monetization records;
- private payout records.
